# Release Notes - v0.1.0

## Overview

Initial bring-up of a custom Raspberry Pi 5 Embedded Linux system based on:

* Linux Kernel
* BusyBox Userspace
* Initramfs Root Filesystem
* BusyBox Init

The system now boots successfully to an interactive shell over UART.

---

## Features Added

### Boot Framework

* Custom `/init` implementation
* Initramfs-based root filesystem
* BusyBox `init` integration
* `/etc/inittab` support

### Filesystem Support

* Added `/etc/fstab`
* Automatic mounting of:

  * `/proc`
  * `/sys`
  * `/dev`
  * `/tmp`

### Startup Infrastructure

* Added `/etc/init.d/rcS`
* Added `/etc/init.d/startup.sh`
* Runtime directory creation:

  * `/var/log`
  * `/var/run`
  * `/var/tmp`

### Console Access

* UART console operational
* BusyBox shell available after boot

## Major Issues Encountered

### Issue #1 - Raspberry Pi 5 Failed To Boot Without Overlay Files

#### Symptom

System failed to boot and no UART output was observed when the overlays directory was removed.

#### Root Cause

Raspberry Pi 5 required:

```text
overlays/bcm2712d0.dtbo
```

for the board revision being used.

#### Resolution

Retained:

```text
overlays/bcm2712d0.dtbo
```

within the boot partition.

---

### Issue #2 - devpts Mount Failure

#### Symptom

Boot log showed:

```text
mount: mounting devpts on /dev/pts failed:
No such file or directory
```

#### Root Cause

After:

```text
devtmpfs -> /dev
```

was mounted, the original:

```text
/dev/pts
```

directory from the initramfs became hidden.

The subsequent `devpts` mount therefore failed because the mount point no longer existed.

#### Resolution

Explicitly recreated the mount point before mounting `devpts`:

```sh
mkdir -p /dev/pts
mount -t devpts devpts /dev/pts
```

---

### Issue #3 - Missing Runtime Directories

#### Symptom

Applications expecting runtime directories failed.

#### Resolution

Added runtime directory creation during startup:

```sh
mkdir -p /var/log
mkdir -p /var/run
mkdir -p /var/tmp
```

inside `rcS`.
