# Release v0.0.1

## Features

* Added boot time reporting during early userspace initialization
* Added startup banner displaying release information
* Improved visibility of the custom `/init` execution sequence

## Validation

### Kernel Boot Log

The primary reference used for validation is the kernel log entry:

```text
[    0.724091] Run /init as init process
```

This indicates that the kernel successfully transitioned to userspace and executed the custom `/init` process approximately **740 ms** after kernel startup.

### Console Measurement

The startup banner also reports a runtime boot-time approximation using `/proc/uptime`.

The value displayed in the console is typically observed around:

```text
0.71s - 0.73s
```

This measurement is not considered the authoritative boot-time metric because `/init` continues executing after the timestamp is collected. However, the difference between the kernel-reported value and the console-reported value is negligible and remains within a few milliseconds.
